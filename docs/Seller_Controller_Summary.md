# Seller Controller — Methods Summary

This document summarizes the public methods in `app/Controllers/Seller.php` for quick reference (paste rows into your spreadsheet).

| Sr No | Page / Module | Description | Technical Details |
|---:|---|---|---|
| 1 | Seller Dashboard | Seller landing page showing products, offers, orders, brands, notifications. | Method: `dashboard()` — Route: `GET /seller/dashboard` — View: `seller/dashboard_new` — Inputs: session (logged_in, user_id) — Outputs: HTML view with `products`, `pending_offers`, `active_orders`, `brands`, `notifications`. |
| 2 | Upload Product (form) | Show product upload form; supports edit mode via `?product_id=` and pre-fills images/bills and taxonomy. | Method: `uploadProduct()` — Route: `GET /seller/upload` — View: `seller/upload_product_new` — Inputs: optional `GET product_id`, session user — Actions: loads `product`, `product_images`, computes `product_listing_type_id` for taxonomy preselect. |
| 3 | Process Upload | Validate and persist new product (or create), upload images (max 7) and bills (up to 2), compute suggested prices, notify admin. | Method: `processUpload()` — Route: `POST /seller/process-upload` (AJAX JSON) — Inputs: multipart files `product_images[]`, `bill_images[]`, form fields (title, description, listing_type, category, prices, etc.) — Validation rules applied — Outputs: JSON {success,message,product_id} — Notes: saves images to `uploads/products/`, bills to `uploads/bills/`, stores `bill_image` as string or JSON array. Also updates posting counters and `products_uploaded_count`. |
| 4 | My Products (listing) | Show seller's products with filters (status, type, category); returns category names when numeric. | Method: `myProducts()` — Route: `GET /seller/products` — View: `seller/products` — Inputs: `GET status,type,category`, session user — Actions: builds product list with primary image join, loads `categories` from DB for filter dropdown, maps `category` numeric IDs to names in results. |
| 5 | Get Product (AJAX) | Return product JSON for editing (ownership check). | Method: `getProduct($id)` — Route: `GET /seller/getProduct/(:num)` — Inputs: product id param, session user — Outputs: JSON {success,data: product} — Fails if not owner. |
| 6 | Update Product (AJAX) | Update basic product fields from modal/form (validate title/description, pricing rules). | Method: `updateProduct()` — Route: `POST /seller/updateProduct` — Inputs: `product_id` and form fields — Validation: title, description, price (if sell) — Outputs: JSON {success,message} — On error returns validation errors. |
| 7 | Delete Product (AJAX) | Delete seller-owned product; removes image and bill files and DB records. | Method: `deleteProduct($id)` — Route: `POST /seller/deleteProduct/(:num)` — Inputs: product id param, session user — Actions: deletes product images (files + `product_images` rows), deletes bill files (parses JSON or single string), deletes product row — Outputs: JSON {success,message} — Logs on errors. |
| 8 | Offers List | Seller's offers (with buyer reliability scores, product info). | Method: `offers()` — Route: `GET /seller/offers` — View: `seller/offers_new` — Inputs: optional `GET status`, session user — Outputs: offers array with aliases (`offered_price`,`offer_type`,`buyer_name`,`buyer_reliability_score`). |
| 9 | Accept Offer | Accept an offer (checks booking conflicts for rentals), creates an order and status history, notifies buyer. | Method: `acceptOffer($offerId)` — Route: `POST /seller/acceptOffer/(:num)` — Inputs: offer id, session user — Actions: conflict checks for rentals, update offer status to `accepted`, create `orders` row, insert `order_status_history`, notify buyer via `notificationModel` — Outputs: JSON {success,message,order_id}. |
| 10 | Reject Offer | Reject an offer with window checks when previously accepted; notify buyer. | Method: `rejectOffer($offerId)` — Route: `POST /seller/rejectOffer/(:num)` — Inputs: offer id, session user — Actions: eligibility checks (24-hour window or before pickup), update status to `rejected`, notify buyer — Outputs: JSON {success,message}. |
| 11 | Orders Listing | Seller's orders with optional status filter. | Method: `orders()` — Route: `GET /seller/orders` — View: `seller/orders` — Inputs: `GET status`, session user — Outputs: orders from `OrderModel->getSellerOrders`. |
| 12 | Create Brand | Create a new brand (seller must meet reliability threshold), handle logo upload. | Method: `createBrand()` — Route: `POST /seller/createBrand` — Inputs: `brand_name`, optional `logo` file — Validation: brand_name required, logo image — Actions: saves logo to `WRITEPATH/uploads/brands/`, inserts brand — Outputs: JSON {success,brand_id}. |
| 13 | Transactions / Seller View | Show seller transaction/Order history with status history. | Method: `transactions()` — Route: `GET /seller/transactions` — View: `seller/transactions` — Inputs: optional `GET type,status,from_date` — Outputs: transactions array enriched with `product_title`, `buyer` info and `status_history`. |
| 14 | Brands List | Seller's brands page with product counts. | Method: `brands()` — Route: `GET /seller/brands` — View: `seller/brands` — Actions: loads brands for seller and computes `product_count` per brand. |
| 15 | Update Brand | Update brand metadata (name, description). | Method: `updateBrand()` — Route: `POST /seller/updateBrand` — Inputs: `brand_id`, `brand_name`, `description` — Ownership check and update — Outputs: JSON {success,message}. |
| 16 | Delete Brand | Delete brand if it has no products. | Method: `deleteBrand($brandId)` — Route: `POST /seller/deleteBrand/(:num)` — Inputs: brand id, session user — Checks: brand ownership, ensure no products with brand_id — Outputs: JSON {success,message}. |
| 17 | Order Details | Show single order details including buyer, product and status history. | Method: `orderDetails($orderId)` — Route: `GET /seller/orderDetails/(:num)` — View: `seller/order_details` — Inputs: order id, session user — Authorization check (seller owns order). |
| 18 | Mark Dispatched | Mark an order as `dispatched` (only from `confirmed`), add status history, notify buyer. | Method: `markDispatched($orderId)` — Route: `POST /seller/markDispatched/(:num)` — Inputs: order id — Actions: validate owner and current status, update order status, insert status history, notify buyer — Outputs: JSON. |
| 19 | Analytics | Seller analytics (totals, month stats, top products, category breakdown, reviews). | Method: `analytics()` — Route: `GET /seller/analytics` — View: `seller/analytics` — Inputs: session user — Actions: queries ProductModel, OrderModel, ReviewModel to compute aggregates — Outputs: stats, `top_products`, `category_stats`. |
| 20 | Profile | Seller profile page. | Method: `profile()` — Route: `GET /seller/profile` — View: `seller/profile` — Inputs: session user — Outputs: user data. |
| 21 | Subscription | Subscription management page. | Method: `subscription()` — Route: `GET /seller/subscription` — View: `seller/subscription` — Inputs: session user. |
| 22 | Notifications | Seller notifications list. | Method: `notifications()` — Route: `GET /seller/notifications` — View: `seller/notifications` — Inputs: session user — Outputs: notifications via `notificationModel->getUserNotifications`. |
| 23 | Help | Static help page for seller. | Method: `help()` — Route: `GET /seller/help` — View: `seller/help`. |
| 24 | Get Listing Types (AJAX) | Returns all listing types. | Method: `getListingTypes()` — Route: `GET /seller/getListingTypes` — Outputs: JSON {success,data: listing_types}. |
| 25 | Get Product Types by Listing (AJAX) | Returns product types for a given listing type id. | Method: `getProductTypesByListing($listingTypeId)` — Route: `GET /seller/getProductTypesByListing/(:num)` — Outputs: JSON {success,data: product_types}. |
| 26 | Get Categories by Product Type (AJAX) | Returns categories for a product type (tolerant to schema differences). | Method: `getCategoriesByProductType($productTypeId)` — Route: `GET /seller/getCategoriesByProductType/(:num)` — Logic: checks `categories` fields for `parent_id` or `product_type_id` fallback — Outputs: JSON {success,data: categories}. |
| 27 | Check Posting Limits (private) | Enforces daily/weekly/monthly posting limits per subscription tier and updates counters. | Method: `checkPostingLimits($user)` (private) — Called by `processUpload()` — Inputs: `$user` data — Actions: consult `posting_limits` table, compute charges, update user counters (`daily_posts_count`, etc.) — Returns: array {allowed:boolean,charge:number,message:string}. |
| 28 | Fitting Alteration Requests | List fitting alteration requests for seller. | Method: `fittingAlterationRequests()` — Route: `GET /seller/fittingAlterationRequests` — View: `seller/fitting_alteration_requests` — Inputs: session user — Outputs: requests joined with product and buyer info. |
| 29 | Respond to Fitting Alteration | Seller responds to alteration requests; updates DB and notifies buyer. | Method: `respondToFittingAlteration($requestId)` — Route: `POST /seller/respondToFittingAlteration/(:num)` — Inputs: `status`, `response` — Ownership check, update row, notify buyer — Outputs: JSON. |
| 30 | Confirm Rental Return | Confirm rental return and mark order `completed`. | Method: `confirmRentalReturn($orderId)` — Route: `POST /seller/confirmRentalReturn/(:num)` — Inputs: order id — Validates ownership and status `returned`, calls `orderModel->updateOrderStatus`, notifies buyer — Outputs: JSON. |
| 31 | Complete Sale Order | Complete a sale order after delivery. | Method: `completeSaleOrder($orderId)` — Route: `POST /seller/completeSaleOrder/(:num)` — Inputs: order id — Validates owner and state, calls `updateOrderStatus`, notifies buyer — Outputs: JSON. |
| 32 | Get Sub-Categories by Category (AJAX) | Returns sub-categories for a category id. | Method: `getSubCategoriesByCategory($categoryId)` — Route: `GET /seller/getSubCategoriesByCategory/(:num)` — Outputs: JSON {success,data: sub_categories}. |

**Common notes**
- Controller file: `app/Controllers/Seller.php`
- Models referenced: `ProductModel`, `ProductImageModel`, `BrandModel`, `OfferModel`, `OrderModel`, `NotificationModel`, `OrderStatusHistoryModel`, `UserModel`, `ReviewModel`.
- File uploads: images saved to `public/uploads/products/`; bills saved to `public/uploads/bills/` (paths stored in DB as `uploads/...`).
- Responses: many mutating actions return JSON for AJAX front-end integration; listing/view methods return server-rendered views.
